行内元素布局笔记

此处所讨论的行内元素包括inline-block

  • 行内元素是不能设定宽高，(inline-block可以).他的高一般由line-hight决定。
  • 行内元素框内有文字和没文字的区别，高度伸展的方向是不一样的。当有文字的时候，高度是向下扩展的，没文字的时候是向上扩展的。
    这条规则是inline元素布局特殊的一点，经常刚开始布局好了之后，在元素之内添加文字导致元素上下乱跳的罪魁祸首。
  • 行内元素垂直对齐的时候常用CSS属性 vertical-align
  • 默认情况下，是子元素的最后一行文字或者是最后一个inline-block与父元素的基线对齐

DOCTYPE

发表于 2016-05-27   |   分类于 HTML

Doctype用法

必须声明在HTML文档的第一行
DOCTYPE并不属于HTML标签。

Doctype作用

  • 指出让解析HTML文档的程序用什么规则来解释文档中的标记
  • 如果文档中的标记不遵循DOCTYPE声明所指定的DTD，可能导致解析HTML文档的程序（通常是浏览器）不能正确的显示HTML文档
  • 如果没有声明DOCTYPE浏览器会使用怪异模式解析执行代码

浏览器的怪异模式与标准模式

  • 怪异模式是浏览器按照自己的标准来解析页面
  • 标准模式是浏览器按照W3C标准来解析页面
    例如CSS的盒子模型为例，怪异模式和标准模式在显示上就不同

Ajax

Ajax - Asynchronous JavaScript and XML 异步JavaScript与XML
Ajax通过JS的XMLHttpRequest对象实现

属性

  • reaadyState
    表示XHR对象的状态：
    值、数值、说明
    UNSENT 0 成功创建XHR对象
    OPENED 1 已调用OPEN方法
    HEADERS_RECEIVED 2 已收到服务器响应头
    LOADING 3 已经收到服务器响应
    DONE 4 响应完成或失败
  • onreadystatechange
    状态改变的事件触发器。readyState属性变化时调用的该属性上的js函数。
  • responseText
    服务器响应的文本
  • responseXML
    服务器响应的XML内容对应的DOM
  • Status
    服务器返回的http状态码。状态码详情见HTTP协议，
  • statusText
    服务器返回状态的文本信息。

方法

  • Open(string method,string url,boolean asynch[,String username,string password])
  • Send(content) content可以是DOM对象，输入流，字符串。如果是异步传输，该方法会被立刻返回。
  • SetRequestHeader(String header,String Value) 设置HTTP请求中的指定头部header的值为value.
    此方法需在open方法以后调用，一般在post方式中使用。
  • getAllResponseHeaders() 返回包含Http的所有响应头信息，其中相应头包括Content-length,date,uri等内容。
    返回值是一个字符串，包含所有头信息，其中每个键名和键值用冒号分开，每一组键之间用CR和LF（回车加换行符）来分隔
  • getResponseHeader(String header) 返回HTTP响应头中指定的键名header对应的值
  • Abort() 停止当前http请求。对应的XMLHttpRequest对象会复位到未初始化的状态。

XHR简单用法

1.open()
2.send() //如果不传数据则必须输入null.在调用send()之后。请求被发送到服务器。如果是异步的则立刻返回，否则等待服务器响应。在收到响应之后，响应的数据自动填充XHR的属性。

* responseText 响应主题被返回的文本。
* responseXML 如果响应的内容类型是text/xml或者application/xml，这个属性中将保存喊着响应数据的XML DOM.
* status 相应的HTTP状态
* statusText HTTP状态的说明

3.收到响应后，第一步检查status属性。在进行下一步操作。

但是如果操作是异步的，则要通过监控readyState属性来操作。每当readyState属性变化时会触发onreadystatechange函数。我们一般只对readyState==4感兴趣。

浏览器差异

IE

对于IE连XHR对象都不一样。另看文章。

opera

  • opera不对readystatechange的触发函数传入一个event对象，所以XHR对象的作用域应该包含readystatechange的作用域。

JS中的对象

属性类型

在JS中有两种属性类型：数据属性，访问器属性。

数据属性

Configurable：能否通过delete删除属性。默认为true.
Enumerable：能否通过for-in循环返回属性。默认为true.
Writable：能否修改属性的值。默认为True.
Value：包含这个属性的数据值。读取和写入属性的时候都是从这个属性来。默认为undefined.

访问器属性

Configurable：能否通过delete删除属性。默认为true.
Enumerable：能否通过for-in循环返回属性。默认为true.
get:在读属性时调用的函数。默认为undefined.
set:在写属性时调用的函数。默认为undefined.
其中get和set不是必须的。

使用Object.getOwnPropertyDescriptor(“object”,”property”)方法取得属性的描述。
返回一个对象包含属性中的对象。

创建对象

工厂模式

构造函数模式

特征：没有显示创建Object,直接赋值给this,没有return.
内部步骤
1.创建一个新对象
2.将构造函数的作用域赋给一个新对象。this指向新对象
3.执行构造函数
4.返回新对象

原型模式

如果不是向prototype中添加方法而是指定一个prototype为一个对象，那么他的constructor属性不再指向原来的constructor需要手动重新指定。

混合模式

这种方式使得每个对象既有自己的属性又可以共享公共方法达到节约内存的目的。

稳妥构造函数模式

稳妥对象指的是没有公共属性，不引用this的对象。适合在安全环境中使用。

正则表达式

正则表达式基本语法

  • ^代表字符串的开头
  • $代表字符串的结尾
  • \b元字符。代表单词的开头或结尾。 \bhi\b 匹配所有的hi
  • \d匹配一位数字。 同[0-9]
  • \w匹配任意字母或者数字或者下划线或者汉字
  • \s匹配任意的空白符

以上匹配符的字母换成大写即可变成反义匹配。如\W变成匹配任意不是字幕、数字、下划线、汉字的字符

  • \B
  • \D
  • \W
  • \S
  • . 匹配除了换行符以外的任何字符。
  • *表示数量 *前面的内容可以连续重复匹配任意次数包括0次。
  • +表示数量 和*不同的是 必须匹配1次或者1次以上
  • ? 匹配0次或者1次

如果想匹配 . * + 这样的符号就必须使用转义符 \

  • {数字}连续匹配n次。 \d{4}-\d{8}匹配电话号。四位区号-八位电话号。{m，n}匹配不少次m次不多于n次。{m,}重复匹配n或者更多次
  • [ ]表示匹配集合中的某一个字符
  • ( )表示分组，相当于子表达式。
  • [^x] 匹配除了x以外的任意字符
  • (^abcd)匹配除了abcd意外的任意字符

练习：解读 ((2[0-4]\d|25[0-5]|[01]?\d\d?)\.){3}(2[0-4]\d|25[0-5]|[01]?\d\d?) 匹配IP地址的正则表达式。

<a[^>]+>匹配用尖括号括起来的以a开头的字符串。
