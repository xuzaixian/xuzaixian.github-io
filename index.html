块格式化上下文

发表于 2016-07-23

定义

在MDN中是这样解释的：
块格式化上下文（block formatting context） 是页面 CSS 视觉渲染的一部分。它是用于决定块盒子的布局及浮动相互影响的一个区域。

下列情况将创建一个块格式化上下文：

  • 根元素或其它包含它的元素
  • 浮动 (元素的 float 不为 none)
  • 绝对定位元素 (元素的 position 为 absolute 或 fixed)
  • 行内块 inline-blocks (元素的 display: inline-block)
  • 表格单元格 (元素的 display: table-cell，HTML表格单元格默认属性)
  • 表格标题 (元素的 display: table-caption, HTML表格标题默认属性)
  • overflow 的值不为 visible的元素
  • 弹性盒子 flex boxes (元素的 display: flex 或 inline-flex)
    块格式化上下文包括了创建该上下文的元素的所有子元素，但不包括创建了新的块格式化上下文的子元素。

个人理解

就相当于隔离出来一个单独的块，这个单独的块里面的布局不会影响到外面的布局。就像JS使用IIFE来创建一个链式的作用域一样。

特性

  • 内部的Box会在垂直方向，从顶部开始一个接一个地放置。
  • Box垂直方向的距离由margin决定。属于同一个BFC的两个相邻Box的margin会发生叠加
  • 每个元素的margin box的左边， 与包含块border box的左边相接触(对于从左往右的格式化，否则相反)。即使存-在浮动也是如此。
  • BFC的区域不会与float box叠加。
  • BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素，反之亦然。
  • 计算BFC的高度时，浮动元素也参与计算。

常用DOM操作

发表于 2016-07-22   |   分类于 HTML

NodeList & HTMLCollection

相同点：都有length属性。 两个类都是实时的，也就是任何动态操作DOM都会实时反映在这两个类中。
不同点

  • NodeList的元素是Node，HTMLCollection的元素是Element。
  • HTMLCollection有一个namedItem方法，可以快速获取元素。

那么假设得到了这个div的子元素构成的HTMLCollection，叫做list，那么使用list.namedItem(“test”)就可以直接得到里面的img元素。

  • HTMLCollection有一个item（index）方法，通过索引返回对象。
  • NodeList通过item(index)使用索引获取对象。

Node & Element

Element继承了node类.
Node不光有element这种节点类型，还有attr属性节点，text文本节点等等。
Element是以HTML标签为单位。而节点则更细致。甚至空格，换行符都可以算作一个文本节点。

选择DOM

  • document.getElementById(String id)
  • document.getElementByTagName(String tagName)
    该方法返回的HTMLCollection是实时的，也就是如果通过该方法获取元素之后，再对DOM树进行操作后，该方法返回的结果也会动态的更新。

  • document.getElementByClassName(String className)
    改方法返回HTMLCollection，该方法不光是document为根元素查找，也可以指定根元素进行查找。

  • document.querySelector(String selector)
    该方法使用深度优先，前序遍历。selector使用CSS选择器的字符串形式。如果没有找到元素返回null。找到多个则返回第一个匹配到的元素。
    使用CSS伪类不会返回任何元素

  • document.querySelectorAll(String selectors)
    该方法使用深度优先，前序遍历，返回选择器所匹配的所有元素。返回类型是NodeList.
    selectors可以是用逗号隔开的多个选择器。
    WebKit 内核的浏览器有一个bug: 如果 selectors参数中包含 CSS伪类 ,返回的elementList包含一个 元素,而不是空的.

创建DOM

  • document.createElement(String tagName)
    该方法传入一个标签名，创建一个相应的元素。
  • document.createTextNode(String string)
    参数string为文本节点的内容。

删除DOM

  • element.remove()
    将该元素从DOM树中删除。（IE不支持,safari移动端不支持）
  • node.removeChild(child)
    node是child的父节点。

添加DOM

  • node.appendChild(dom)
    dom一般为document.createElement()创建出来的节点。插入到兄弟节点的最后。
  • node.insertBefore()

如果referenceElement为null则newElement将被插入到子节点的末尾。如果newElement已经在DOM树中，newElement首先会从DOM树中移除。

操作Class

  • element.classList
    该方法是一个只读属性，返回一个DOMTokenList，包含所有Class属性。
    该方法返回的对象具有如下几个方法

  • element.className
    返回一个字符串，多个class属性由空格分隔。
    IE浏览器不支持第二个参数

操作ID

  • element.id;
    该属性返回当前节点的id。也可以通过设置该属性来设置该节点的ID值。

获取标签名

  • element.tagName
    该属性返回该节点的标签名。对于元素节点来说该属性和nodeName属性是相同的。

属性操作

  • element.getAttribute(String)
    该方法返回指定属性的值，如果指定属性不存在则按标准来说返回null,有些返回””.
  • element.hasAttribute(String)
    该方法放回一个布尔值，指示是否有该属性。
  • element.removeAttribute(String)
    删除指定属性。
  • element.setAttribute(String,String)
    添加一个新的属性。或改变已有属性的值。

后代操作

  • element.children
    该只读属性返回一个当前元素的子元素HTTPCollection
  • element.children.length
    返回有多少个后代元素。
  • element.firstElementChild
    获取第一个子元素
  • element.lastElementChild
    获取最后一个子元素

操作节点内容

  • element.innerHTML
    该属性返回该节点内部（不包括自身)的所有内容以HTML代码形式表现出来，也就是包括HTML标签。
  • element.outerHTML
    该属性返回该节点（包括自身)的所有内容以HTML代码形式表现出来，也就是包括HTML标签。

样式编辑

  • element.style
    使用该属性获取或设置css样式都是内嵌样式，也就是在HTML标签中的style属性中设置。

  • window.getComputedStyle(element,[pseudoElt])
    element 指定的元素
    pseudoElt 指定一个伪元素进行匹配。对于常规的元素来说可省略。
    该方法返回当前元素所有最终使用的CSS属性值。返回的是一个CSS样式声明对象([object CSSStyleDeclaration])。READ ONLY。
    使用CSSStyleDeclaration的getPropertyValue（String）方法返回指定的CSS属性的值，参数传入字符串形式的CSS属性名。

  • element.currentStyle
    改属性是IE特有的一个属性，用来返回该对象的CSS样式声明对象。
    对于IE6~8需要使用getAttribute方法访问CSS样式对象中的属性。

CSS动画

发表于 2016-07-22

transition

使用transform属性定义动画主要是定义简单动画。
优点：简单易用，定义简单动画的时候首选。
缺点

  • 只能定义动画的起始形态与结束形态，中间过程不可控
  • 而且每一个属性的动画都要单独写一条语句,或者使用matrix()方法
  • 只能播放一次。
  • 不能暂停/继续

元素的2D/3D变换属性

以下属性需要配合transform属性一起使用。

  • translate(x,y) 根据当前位置移动x,y的坐标。允许负值。
  • rotate(angle) 元素顺时针旋转deg角度（单位是角度），允许负值。
  • scale(x,y) 元素尺寸增加或缩小x,y倍。
  • skew(x-angle,y-angle) 元素倾斜x°，y°。倾斜的默认原点是元素的中心点。就像把一个长方形倾斜成平行四边形那样倾斜。
  • rotateX() 沿X轴做3D旋转
  • rotateY() 沿Y轴做3D旋转
  • transform-origin(x,y) 改变旋转元素的X,Y轴或者说是中心点。x预设值有left center right,y预设值有top center bottm.单位可以使用绝对长度或者%.
  • transform-style 子元素是否应用3D变换。默认flat，不保留，preserve-3d属性为保留。

transition属性应用动画

这样即可使变化的css属性以动画的形式表现出来。如果不规定时间则不会有动画效果，因为时间默认是0.
效果开始于指定的 CSS 属性改变值时。

以上CSS属性会使当鼠标悬停在div上面时，使DIV宽度以动画的形式从100PX变长到300PX。
对于使用颜色的属性，transitions同样适用。

##transition属性

  • transition: 简写属性，包含四个属性。transition-property transition-duration transition-timing-function transition-delay。
  • transition-property: 使用动画的CSS属性。
  • transition-duration：动画的过度时间。
  • transition-timing-function: 速度曲线，默认是’ease’,还有几个预设值，linear 规定以相同速度开始至结束的过渡效果（等于 cubic-bezier(0,0,1,1)）。
    ease 规定慢速开始，然后变快，然后慢速结束的过渡效果（cubic-bezier(0.25,0.1,0.25,1)）。
    ease-in 规定以慢速开始的过渡效果（等于 cubic-bezier(0.42,0,1,1)）。
    ease-out 规定以慢速结束的过渡效果（等于 cubic-bezier(0,0,0.58,1)）。
    ease-in-out 规定以慢速开始和结束的过渡效果（等于 cubic-bezier(0.42,0,0.58,1)）。
    cubic-bezier(n,n,n,n) 自定义速度曲线（三次卡塞尔曲线）。
  • transition-delay: 动画开始前等待时间。

animation

使用animation创建动画需要@keyframes定义关键帧配合使用。
优点

  • 可以定义多个关键帧
  • 可以在一个keyframes中同时定义多个属性的变化动画。
  • 可以暂停/继续播放动画
  • 可以循环播放动画
  • 可以规定动画播放结束之后是否逆向播放动画使元素回到动画之前的状态。

使用@keyframes定义动画的时候同时需要定义一个动画的名称。
定义keyframes关键帧的时候可以使用%来规定在动画运行到百分之多少的时候关键帧变化成什么样。或者使用from和tu关键词，分别代表0%,100%。
应用@keyframes的时候只需要在animation属性上应用@keyframes的名称即可

animation属性

animation: 简写属性 设置6个属性

  • animation-name
  • animation-duration
  • animation-timing-function
  • animation-delay
  • animation-iteration-count
  • animation-direction

animation-name: @keyframes的动画名称
animation-duration: 规定动画完成的一个周期所用的时间
animation-time-function:预设值与默认值同transition
animation-delay： 动画延迟多久之后开始
animation-iteration-count:播放次数
animation-direction:默认值normal，alternate动画应该轮流反响播放。

行内元素布局笔记

Doctype用法

必须声明在HTML文档的第一行
DOCTYPE并不属于HTML标签。

Doctype作用

  • 指出让解析HTML文档的程序用什么规则来解释文档中的标记
  • 如果文档中的标记不遵循DOCTYPE声明所指定的DTD，可能导致解析HTML文档的程序（通常是浏览器）不能正确的显示HTML文档
  • 如果没有声明DOCTYPE浏览器会使用怪异模式解析执行代码

浏览器的怪异模式与标准模式

  • 怪异模式是浏览器按照自己的标准来解析页面
  • 标准模式是浏览器按照W3C标准来解析页面
    例如CSS的盒子模型为例，怪异模式和标准模式在显示上就不同

Ajax

Ajax - Asynchronous JavaScript and XML 异步JavaScript与XML
Ajax通过JS的XMLHttpRequest对象实现

属性

  • reaadyState
    表示XHR对象的状态：
    值、数值、说明
    UNSENT 0 成功创建XHR对象
    OPENED 1 已调用OPEN方法
    HEADERS_RECEIVED 2 已收到服务器响应头
    LOADING 3 已经收到服务器响应
    DONE 4 响应完成或失败
  • onreadystatechange
    状态改变的事件触发器。readyState属性变化时调用的该属性上的js函数。
  • responseText
    服务器响应的文本
  • responseXML
    服务器响应的XML内容对应的DOM
  • Status
    服务器返回的http状态码。状态码详情见HTTP协议，
  • statusText
    服务器返回状态的文本信息。

方法

  • Open(string method,string url,boolean asynch[,String username,string password])
  • Send(content) content可以是DOM对象，输入流，字符串。如果是异步传输，该方法会被立刻返回。
  • SetRequestHeader(String header,String Value) 设置HTTP请求中的指定头部header的值为value.
    此方法需在open方法以后调用，一般在post方式中使用。
  • getAllResponseHeaders() 返回包含Http的所有响应头信息，其中相应头包括Content-length,date,uri等内容。
    返回值是一个字符串，包含所有头信息，其中每个键名和键值用冒号分开，每一组键之间用CR和LF（回车加换行符）来分隔
  • getResponseHeader(String header) 返回HTTP响应头中指定的键名header对应的值
  • Abort() 停止当前http请求。对应的XMLHttpRequest对象会复位到未初始化的状态。

XHR简单用法

1.open()
2.send() //如果不传数据则必须输入null.在调用send()之后。请求被发送到服务器。如果是异步的则立刻返回，否则等待服务器响应。在收到响应之后，响应的数据自动填充XHR的属性。

* responseText 响应主题被返回的文本。
* responseXML 如果响应的内容类型是text/xml或者application/xml，这个属性中将保存喊着响应数据的XML DOM.
* status 相应的HTTP状态
* statusText HTTP状态的说明

3.收到响应后，第一步检查status属性。在进行下一步操作。

但是如果操作是异步的，则要通过监控readyState属性来操作。每当readyState属性变化时会触发onreadystatechange函数。我们一般只对readyState==4感兴趣。
